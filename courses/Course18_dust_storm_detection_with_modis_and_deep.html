Data Acquisition and Preprocessing
10 min

Paste the following code into your GEE code editor and click Run. Read the beginner notes below to understand what each line does.

Data Acquisition and Preprocessing
// Define the region of interest (North Morocco).
var roi = ee.Geometry.Rectangle([-6, 32, -2, 35]);

// Define the date range.
var startDate = '2023-07-01';
var endDate = '2023-07-31';

// Load the MODIS surface reflectance dataset.
var modis = ee.ImageCollection('MODIS/061/MOD09GA')
    .filterDate(startDate, endDate)
    .filterBounds(roi);

// Function to calculate NDVI.
var calculateNDVI = function(image) {
  return image.normalizedDifference(['sur_refl_b02', 'sur_refl_b01']).rename('NDVI');
};

// Apply the NDVI calculation to the MODIS collection.
var modisWithNDVI = modis.map(calculateNDVI);
🔑
Beginner note: This code block first defines the region of interest (ROI) as a rectangle in North Morocco and sets the time frame for data acquisition. Then, it loads the MODIS surface reflectance dataset from Google Earth Engine, filtering it by date and location. A function is defined to calculate the Normalized Difference Vegetation Index (NDVI) using the near-infrared (band 2) and red (band 1) bands. Finally, this function is applied to the MODIS image collection to add the NDVI band to each image.

Dust Storm Detection and Visualization
10 min

Add this block after the previous code. It continues the analysis with additional processing and visualization.

Dust Storm Detection and Visualization
// Composite the MODIS collection to a single image (median).
var modisImage = modisWithNDVI.median().clip(roi);

// Define a threshold for NDVI to identify potential dust areas.
var ndviThreshold = 0.1;

// Create a mask for dust storms based on the NDVI threshold.
var dustMask = modisImage.select('NDVI').lt(ndviThreshold);

// Create a false-color composite for visualization.
var falseColor = modisImage.select(['sur_refl_b04', 'sur_refl_b03', 'sur_refl_b01']).multiply(0.0001);

// Display the false-color composite on the map.
Map.addLayer(falseColor, {min: 0, max: 0.3}, 'False Color Composite');

// Display the dust storm mask on the map.
Map.addLayer(dustMask.updateMask(dustMask), {palette: 'red'}, 'Dust Storms');
🔑
Beginner note: This block begins by creating a median composite image from the MODIS collection to reduce cloud cover. It then sets an NDVI threshold to identify dust storms (lower NDVI values often indicate less vegetation and potentially dust). A mask is created based on this threshold. A false-color composite is created using specific MODIS bands (5,4,3) to visually highlight dust. Finally, both the false-color composite and the dust storm mask are added to the map for visualization. The red color highlights areas identified as dust storms.

Export Results to Google Drive
5 min

Add this block at the end of your script to export the final map as a GeoTIFF to your Google Drive. After clicking Run, go to the Tasks tab in GEE and click Run next to each export task.

Export to Google Drive
// Export the dust storm mask to Google Drive.
Export.image.toDrive({
  image: dustMask,
  description: 'Dust_Storm_Mask',
  folder: 'GEE_Exports',
  region: roi,
  scale: 1000,
  maxPixels: 1e13
});
🔑
Beginner note -- Export.image.toDrive(): This function does not export immediately. It creates a task in the GEE Tasks tab that you must manually start by clicking Run. The file will appear in your Google Drive folder (GEE_Courses) as a GeoTIFF that you can open in QGIS or ArcGIS.
